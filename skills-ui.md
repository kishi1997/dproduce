# UI / Design Skills

## Goal

- Elevate wireframes to production-level LP design
- Overly simple UI is forbidden
- Always maintain a "designed" appearance

---

## Layout

- Create clear visual distinction between sections:
  - Background color
  - Spacing
  - Borders and decorative elements
- Monotonous vertical stacking is forbidden
- Be conscious of visual flow (Z-pattern or F-pattern)

---

## Section Design

- Give each section a clear role
- Always include the following:
  - Title
  - Description text
  - Content (cards, images, etc.)

---

## Section Title

- Base the design on the top page
- Create a shared class for lower-level pages and use it
- Maintain the structure: English title + Japanese subtitle

---

## Component Reuse

- Always reference existing home.php and other pages
- Reuse the same UI whenever possible
- Only change the text content
- Minimize new component creation

---

## Feature Section

- Simple lists are forbidden
- Use card UI
- Add visual contrast through hover effects, spacing, and hierarchy

---

## Typography

- Specify font sizes in px
- Define a clear hierarchy:
  - Heading > Subtitle > Body text
- Use appropriate line height and spacing

---

## Spacing

- Treat spacing as part of the design
- Use generous spacing between sections
- Create rhythm in spacing between elements

---

## Image Usage

- Use free stock images
- Integrate images naturally as part of the layout
- Simply placing images without intent is forbidden

---

## Visual Quality

- Flat, undesigned UI is forbidden
- Always include:
  - Contrast
  - Hierarchy
  - Visual flow
- Completely eliminate any wireframe-like appearance

---

## Anti-Patterns (Forbidden)

- Text-only sections
- Layouts with no spacing
- All sections having the same design
- UI that looks like a default/unstyled state

## Image Acquisition

- Obtain images from one of the following:
  - Free stock photo sites
  - AI-generated images (e.g., Google)

- Do not place images arbitrarily — use them as part of the design
