# Lower Page UI Skills

## Goal

- Organize information clearly and make it easy to understand
- Prioritize readability and trustworthiness

---

## Layout

- Too simple is not acceptable, but over-decoration is also not acceptable
- Prioritize readability

---

## Section

- Organize content by information type
- Make grouping clear and explicit

---

## Feature

- Organize using card UI
- Be conscious of comparability and scannability

---

## Consistency

- Match the design tone with the top page

---

## CTA

- Place only where necessary
- Do not overuse

## First View (FV)

- Lower-level pages must use the shared FV design
- Only the following may be changed:
  - Text
  - Image
- Do not change the layout or structure

---

## Page Structure

- Wrap the outer element with a slug class (e.g., p-price)
- Scope design within the page unit
